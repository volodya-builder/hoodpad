# hood — три идеи роста (детальная проработка)

Статус: черновик для обсуждения, 20.07.2026. Порядок внедрения: №3 → №1 → №2 (от нулевого риска к большому релизу).

---

## Идея 3. «Подушка выкупа» — floor-бейдж (чистый фронтенд, 1–2 дня)

**Суть.** Казна уже выкупает токены, но это спрятано во вкладке Казна. Делаем поддержку казны видимой на каждом шагу — это готовое уникальное преимущество, которое сейчас не работает на маркетинг.

**Что показываем.** «Подушка» токена = суммарный ETH, потраченный казной на выкуп этого токена, плюс сожжённый сапплай. Все данные уже есть в subgraphTreasuryOps() и событиях казны — нужна только агрегация per-token.

**UI.**
- Бейдж на карточке токена: «🛡 Выкуп: 0.4 ETH».
- Сортировка/фильтр на главной: «по поддержке казны».
- Блок на странице токена: «Казна в этом токене: потрачено X ETH · сожжено Y (Z% сапплая)». Отметки выкупов на графике уже есть.
- Общий счётчик на сайте (футер или главная): «Казна вернула рынку: X ETH» — постоянно растущее число, главная метрика бренда.

**Маркетинг.** Позиционирование «единственный лончпад, где за токенами лежат реальные деньги». Еженедельный твит с ростом счётчика. Для агрегаторов (GMGN и т.п.) это уникальная метрика, которой нет ни у кого.

**Риски.** Нет. Контракты не трогаем.

## Идея 1. Он-чейн рефералка — пожизненный процент с приведённых трейдеров (без изменения основных контрактов)

**Суть.** Каждый пользователь получает реф-ссылку. Все, кто пришёл по ней и торгует, пожизненно приносят рефереру долю со своих комиссий. Аудитория пиарит платформу, потому что это её кэшфлоу.

**Экономика.** Комиссия трейда 1% делится 50/20/30. Рефералку финансируем из командных 20%: отдаём рефереру 25% командной доли = 0.05% объёма приведённого трейдера. Пример: привёл кита с объёмом 100 ETH/мес → комиссий 1 ETH → команде 0.2 ETH → рефереру 0.05 ETH/мес пассивно. Казна и доля создателей не затронуты — «hood возвращает» не страдает.

**Механика привязки.**
- v0 (быстро): ссылка вида hoodandarrow.com/#/r/КОД → код в localStorage → при первом трейде фронт пишет пару {трейдер → реферер} в Firebase. Расчёт начислений по Goldsky (объёмы трейдера × ставка).
- v1 (доверие): лёгкий immutable Referral-контракт с функцией register(referrer) — одна дешёвая транзакция при первом трейде, привязка видна в блокчейне, накрутить задним числом нельзя. Деплой не трогает фабрику/пулы/казну.

**Выплаты.** Начисления копятся в дашборде; выплата раз в неделю батчем — владелец запускает локальный скрипт (как auto-claim в админке), приватный ключ никуда не уходит. Минимальный порог выплаты 0.005 ETH.

**Анти-абьюз.**
- Самореферал = фактически кэшбек 5% от собственной комиссии. Не опасно: комиссия всё равно уплачена полностью, теряет только командная доля частично.
- Wash-trading ради рефералки невыгоден: платишь 1%, возвращаешь 0.05%.
- Реферер активен только если сам сделал ≥1 трейд (отсекает ботов-спамеров).

**UI.** Страница «Заработать»: своя ссылка, счётчики (кошельков приведено, их объём, начислено/выплачено), лидерборд рефереров — сам по себе виральный контент.

**Работа.** v0 — 2–3 дня (фронт + Firebase + скрипт расчёта). v1 — плюс простой контракт (Slither, тестнет).

## Идея 2. Со-создатели — первые покупатели делят комиссии токена (флагман v2, нужен новый контракт)

**Суть.** Создатель при запуске может отдать часть своей 50%-доли первым N покупателям: «первые 50 холдеров становятся со-создателями и пожизненно делят 25% комиссий токена». Ранний покупатель зарабатывает не только на росте цены, но и на каждой будущей сделке — ему выгодно не дампить, а шиллить токен годами. Каждый токен рождается с армией мотивированных маркетологов.

**Параметры запуска (выбирает создатель, опционально).**
- N слотов со-создателей: 25 / 50 / 100.
- Доля, отдаваемая им: например половина создательской (25% всех комиссий токена).
- Минимальная покупка для слота: например 0.01 ETH (защита от пыли).

**Распределение внутри пула со-создателей.** Pro-rata по ETH первой покупки с капом ~10% пула на кошелёк. Почему не поровну: при равных долях выгодно сибилить (разбить покупку на 50 кошельков и занять все слоты). При pro-rata сибил бессмыслен — доля пропорциональна вложенному, кап режет китов.

**Реализация.** Новый контракт CoCreatorSplitter: creator-доля комиссий идёт в него, он регистрирует первых N покупателей и даёт каждому claim(). Пул должен сообщать сплиттеру о покупках → нужна новая версия BondingCurvePool и фабрики (v2). Старая фабрика продолжает работать параллельно, существующие токены не затронуты.

**FOMO-механика в UI.** На карточке токена: бейдж «Co-op» и прогресс «Со-создатели: 34/50 мест». Живой таймер захвата слотов продаёт себя сам. В профиле — «мои доли со-создателя: токен, %, накоплено, claim».

**Экономика создателя.** Опция добровольная: жадный оставит все 50% себе, щедрый получит армию шиллеров. Рынок быстро покажет, что co-op токены растут лучше — дальше механика распространится сама.

**Риски и путь.** Новый контракт = обязательный прогон Slither, полный цикл на тестнете (chainId 46630), затем витринный co-op запуск на mainnet силами команды. Это большой релиз уровня «hood v2» — под него же можно упаковать launch-тред и заявку в GMGN.

**Слоган.** «На других лаунчпадах ты выходишь из токена. На hood токен платит тебе».
